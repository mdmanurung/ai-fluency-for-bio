Syllabus

Logistics

  • Format: self-paced. No instructor, no schedule, no grades.
  • Suggested pacing: ~1 week per unit, ~3–5 hours per week (Weeks 1–2) or ~5–7 hours per week (Weeks 3–4). Allow yourself 4–8 weeks end-to-end depending on background and time.
  • Cadence: each week page is self-contained — readings → practice → knowledge check → project. Move at the speed you learn.
  • Duration scope: this is a foundations course; real fluency develops over months of supervised practice on real research problems. Plan accordingly.

Prerequisites

  • Working comfort with R or Python at a scripting level (read and modify a 100-line script).
  • An LLM chat account — free tier of Claude, ChatGPT, or Gemini works for most exercises (see budget note below).
  • A GitHub account.
  • Laptop with terminal access; ability to install Quarto and a code assistant.
WarningFree-tier budget note

Free-tier LLMs work for Week 1–2 chat exercises. The Week 3 mini-project and Week 4 final project benefit from a paid tier (Claude Pro / ChatGPT Plus) or institutional API access — free-tier rate limits will otherwise interrupt sustained code-assistance sessions. If you have institutional access via a research computing service, check whether they offer enterprise LLM access before paying out of pocket.

Required tools

  • Quarto (current stable; ≥ 1.5) — for reproducible reports.
  • Git + GitHub — for version control and storing your project artifacts.
  • A coding assistant (one of): Claude Code, Cursor, or VS Code + GitHub Copilot.
  • A chat LLM (one of): Claude, ChatGPT, or Gemini. Course examples use Claude by default.
  • A grounded literature tool (one of): Elicit, Consensus, SciSpace, or Perplexity (sources mode).
  • Bioinformatics environment: Python with Scanpy is the course default for the hands-on track. The 5-module scRNA-seq pipeline runs in Google Colab out of the box (no install required for Modules 3–5; Modules 1–2 need HPC or are skippable). If you prefer a local setup, a pinned environment (conda or uv) is supported — see Week 3. R/Seurat users can substitute the equivalent Bioconductor toolchain for the mini-project; pointers in Code assistance.

Course structure

The course covers three conceptual tracks and a 5-module scRNA-seq pipeline across four self-paced units:

Unit Theme Hands-on Project (with self-rubric)
1 Foundations & the 4 D’s Tooling setup; workflow audit Reflection (≤ 500 words) + tooling confirmation
2 LLM literacy Solo prompt clinic Prompt-engineering exercise (weak vs. strong, with critique)
3 scRNA-seq I — QC, normalisation, clustering (Modules 3 & 4) Module 3 + Module 4 in Colab Mini-project: reproducible AI-assisted PBMC 3k QC + clustering report
4 scRNA-seq II — annotation, final project (Module 5) Module 5 in Colab Final project: literature brief, analysis, or protocol — your choice

Modules 1 (FastQC) and 2 (Cell Ranger / STARsolo) are reading material before Week 3 if you want the full picture. Module 2 needs HPC; if you don’t have it, skip directly to Module 3 — a pre-built count matrix is loaded by Scanpy.

Self-assessment

This course has no grades. Instead, you assess yourself with four kinds of feedback:

Mechanism Where it lives What it’s for
Per-page Check your understanding Bottom of each conceptual page Confirm a single page landed before moving on (3–5 questions, worked answers)
Per-week Knowledge check End of each week page Cumulative check across the week’s readings (5–8 mixed conceptual + applied questions)
Per-week Hands-on practice Middle of each week page Apply the unit to your own data / questions (2–3 small exercises with self-check answers)
Project self-rubric End of each week page (Weeks 1, 2, 3, 4) Grade your own project against the same dimensions an instructor would

If you miss two or more knowledge-check questions on a topic, revisit the reading and redo the practice. The disclosure rubric below applies to every project — it is not graded by anyone, but if you cannot defend the disclosure, the project is not finished.

Disclosure rubric

Every project includes a disclosure section, scored on these dimensions (use it as a self-check on whether your work meets the bar):

Dimension 0 1
Tools listed None or vague (“used AI”) Specific tool, version/model, and access tier (e.g., “Claude Sonnet 4.6, Pro tier, web UI”)
Use described Generic (“for help”) Concrete uses (“drafted DESeq2 scaffold; rewrote intro paragraph; suggested 3 papers, 2 verified, 1 fabricated”)
Verification stated Not mentioned Names what you checked and how (ran tests, verified citations on PubMed, ran code on test data)
Rejections noted Not mentioned Lists at least one AI suggestion you rejected and why

A template disclosure paragraph is provided with each project.

AI-use policy

This is a course about using AI tools well. You are expected to use them on most exercises. Following pedagogical research, two projects include an AI-free or AI-second baseline component so you can develop and demonstrate the underlying skill, not just the collaboration skill:

  • Week 1 reflection: write your initial classification of three of your own tasks without AI consultation. Then, optionally, run them past an AI and note what changed.
  • Week 3 mini-project: write a small (~25-line) AI-free baseline of the PBMC 3k QC pass first, by hand — sc.datasets.pbmc3k(), mt-gene flag, three QC violin plots, threshold filter, log-normalise. Then iterate with an AI assistant on the rest (clustering, marker checks, refactor). Submit both, with a short note on what the AI changed and why you accepted or rejected each change.

For all projects:

  • Disclose which tools you used and how. See the disclosure rubric above.
  • Verify every factual claim, citation, and piece of code before relying on it.
  • Own your output: you are the author of record. AI-generated text used verbatim must be quoted or marked.
  • Protect sensitive data: see the data policy below.

Data policy (required reading before any exercise using real data)

Do not paste any of the following into a third-party LLM service without explicit, documented permission and an approved data-handling pathway:

  • Patient-identifiable data or any PHI under HIPAA (US) / personal data under GDPR (EU/UK).
  • Controlled-access data (NIH dbGaP, UK Biobank, EGA, etc.) — controlled-access agreements typically prohibit third-party transmission.
  • BAA-restricted clinical data — even de-identified data may be subject to your institution’s Business Associate Agreements.
  • Embargoed pre-publication data — including unpublished collaborator data, manuscripts under review, and grant proposals.
  • Sequence or other data under a Material Transfer Agreement (MTA) with restrictive terms.

For any such data: use on-premises, institutional, or vendor enterprise tiers with appropriate contracts (e.g., zero-retention with a signed BAA where applicable). When in doubt, consult your IRB / data steward / institutional privacy office before the exercise, not after.

The course dataset (10x PBMC 3k) is public and safe to use with any tool. Supplementary references to other public datasets (e.g., from GEO or ArrayExpress) are also safe.

Recommended readings

Two short readings per unit is the assigned load; the rest is a menu.

Foundations / fluency

  • Anthropic, AI Fluency: Framework & Foundations — the source of the 4 D’s framework. Free course materials and PDF available from Anthropic.
  • Mollick, E. (2024). Co-Intelligence: Living and Working with AI. Portfolio. — practical framing for working with LLMs.

How LLMs work and why they fail

  • Bender, E. M., Gebru, T., McMillan-Major, A., & Shmitchell, S. (2021). On the dangers of stochastic parrots: Can language models be too big? 🦜 FAccT ’21. Foundational; pre-RLHF era.
  • Wei, J., et al. (2022). Emergent abilities of large language models. TMLR. — Read with the critique below.
  • Schaeffer, R., Miranda, B., & Koyejo, S. (2023). Are emergent abilities of large language models a mirage? NeurIPS. — Important counterpoint to Wei et al.
  • Ouyang, L., et al. (2022). Training language models to follow instructions with human feedback. NeurIPS. — InstructGPT / RLHF.
  • Huang, L., et al. (2023). A survey on hallucination in large language models: Principles, taxonomy, challenges, and open questions. ACM TOIS. — Useful taxonomy: factuality vs. faithfulness hallucination.

AI in biology / science

  • Bunne, C., et al. (2024). How to build the virtual cell with artificial intelligence: Priorities and opportunities. Cell 187(25), 7045–7063. — Forward-looking perspective; advanced.
  • Boiko, D. A., MacKnight, R., Kline, B., & Gomes, G. (2023). Autonomous chemical research with large language models. Nature 624, 570–578. — Chemistry rather than biology; useful as analogy for autonomous research workflows.

Ethics and policy

  • Kay, J., Kasirzadeh, A., & Mohamed, S. (2024). Epistemic injustice in generative AI. AAAI/ACM AIES. — Frames the epistemic risks of generative AI for shared knowledge.
  • ICMJE — current Recommendations on AI in authorship (read the current online version, not a static cite).
  • Nature — current editorial policies on AI (current online version).
  • Science — current AI policies via the journal family’s editorial guidelines (current online version).
NoteA note on the reading list

Citations were verified at the time of writing (April 2026). Editorial policies and “current best” surveys date quickly — confirm before relying on them. Items not flagged as core are still valuable supplements; the assigned load is two short readings per unit.

Course policies

  • Pacing: there are no deadlines. The “weeks” are recommended pacing milestones, not a schedule.
  • Working with peers (optional): if you take this with a study group, share verification approaches and prompt logs but write your own work. Use the self-review template on each other’s projects.
  • AI collaboration: governed by the AI-use policy above. Disclosure is the bar.

Academic integrity

If you are taking this course as part of a programme that submits credit, the same standard applies as in any course: AI output submitted as your own unverified, undisclosed work — without verification or critical engagement — violates academic integrity, regardless of how the text was produced. The disclosure rubric makes the expectation concrete.

If you are taking the course on your own, the same standard still applies — to yourself. Disclosure and verification are habits to build now, not bureaucracy to perform later.

Accessibility

Materials are designed to be screen-reader friendly. Please report accessibility issues via the course repository.
